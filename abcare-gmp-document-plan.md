# ABCare GMP 문서 작성 계획

ABCare 디지털치료제 앱의 식약처 인증을 위한 GMP 문서 작성 계획.

---

## 1. 문서 개요

### 1.1 작성 대상 제품

| 항목 | 내용 |
|------|------|
| 품목명 | 병원진료용 소프트웨어 (예상) |
| 제품명 | ABCare |
| 제품 유형 | 디지털치료제 (DTx) |
| 플랫폼 | 모바일 앱 (iOS/Android) |

### 1.2 주요 기능

1. **복약관리 (Medication Management)**
   - 처방전 기반 복약 알림
   - 복용 기록 추적
   - 복약 준수율 모니터링

2. **통증기록 (Pain Recording)**
   - 일상적 통증 수준 기록
   - 통증 부위 및 강도 추적
   - 추이 분석 및 시각화

3. **처방전관리 (Prescription Management)**
   - 처방전 정보 조회
   - 투약 일정 관리
   - 의료진 연동

4. **마음약국/CBT (Cognitive Behavioral Therapy)**
   - 인지행동치료 콘텐츠 제공
   - 심리 상태 자가평가
   - 치료 프로그램 진행 관리

## 3. Anthony 담당 문서: SRS (요구사항 명세서)

### 3.1 SRS 문서 구조 (Canmore 기준)

```
0. 제·개정 이력표
1. 설계 대상
   1.1 제품명
   1.2 사용목적
   1.3 작용원리
   1.4 등급분류
   1.5 유사제품 정보
2. 기능 요구사항
   2.1 ABCare Patient App
   2.2 ABCare Medical Web (해당 시)
   2.3 유사 제품으로 도출된 요구사항
   2.4 기타 요구사항
3. 검토 의견
4. 결론
```

### 3.2 ABCare Patient App 요구사항 ID 체계

Canmore의 `SRS-P001` 형식을 따름:

```
SRS-P001 ~ SRS-P0XX : Patient App 요구사항

기능별 그룹:
- SRS-P001 ~ P010 : 회원가입 및 로그인
- SRS-P011 ~ P020 : 홈 및 대시보드
- SRS-P021 ~ P040 : 복약관리
- SRS-P041 ~ P060 : 통증기록
- SRS-P061 ~ P080 : 처방전관리
- SRS-P081 ~ P100 : 마음약국/CBT
- SRS-P101 ~ P120 : 마이페이지/설정
- SRS-P121 ~ P140 : 알림 및 공통기능
```

### 3.3 작성 예시 (복약관리)

```markdown
[SRS-P021] 복약 일정 조회
사용자는 처방된 약물의 복용 일정을 조회할 수 있어야 한다.
- 약물명, 복용량, 복용 시간 정보 표시
- 일별/주별/월별 일정 보기 제공
- 완료/미완료 상태 구분

[SRS-P022] 복약 알림
사용자에게 복용 시간에 푸시 알림을 제공해야 한다.
- OS별 푸시 알림 지원 (iOS/Android)
- 알림 소리 및 진동 설정 가능
- 알림 스누즈 기능 제공

[SRS-P023] 복용 기록
사용자는 약물 복용 여부를 기록할 수 있어야 한다.
- 복용 완료 버튼 클릭 시 기록
- 복용 시간 자동 기록 또는 수동 입력
- 미복용 사유 입력 (선택)
```

## 4. Anthony 담당 문서: SDF (상세 설계 파일)

### 4.1 SDF 문서 구조 (Canmore 기준)

```
0. 제·개정 이력표
1. 개요
   1.1 목적
   1.2 개발 제품
   1.3 개발 목표 및 수행 인원
2. 용어정의
3. 소프트웨어 아키텍처 설계도
   3.1 소프트웨어 아키텍처 구성 및 설명
   3.2 Patient App Client 화면 구성 및 주요 기능
   3.3 API Server 주요 기능
4. 소프트웨어 설계 기술서
   4.1 Patient App Client
      4.1.1 회원가입 및 로그인
      4.1.2 홈
      4.1.3 복약관리
      4.1.4 통증기록
      ...
```

### 4.2 SDS ID 체계

SRS ID와 매핑되는 설계 ID:

```
SDS-P001 ~ : Patient App 설계
SDS-A001 ~ : API Server 설계

매핑 예시:
SRS-P021 (복약 일정 조회)
  → SDS-P030 MedicationSchedule.render()
  → SDS-P031 fetchMedicationSchedule()
  → SDS-A050 MedicationService.getSchedule()
```

### 4.3 설계 기술서 작성 형식

| SRS ID | 주요 기능 | 기능 설명 | SDS-ID | 함수명 | 함수 기능 설명 |
|--------|----------|----------|--------|--------|---------------|
| SRS-P021 | 복약 일정 조회 | 처방된 약물의 복용 일정을 조회 | SDS-P030 | MedicationSchedule.render() | 복약 일정 화면 표시 |
| | | | SDS-P031 | fetchMedicationSchedule() | 서버에서 일정 데이터 조회 |
| | | | SDS-P032 | formatScheduleData() | 일정 데이터 포맷팅 |
