## 최소 권장사양 산정 근거

### 1. "최소 권장사양"의 의미

| 구분 | 설명 |
|------|------|
| **최소 지원 (Minimum)** | 앱 설치 및 실행이 **기술적으로 가능한** 최소 버전 (코드 설정) |
| **최소 권장 (Recommended)** | **쾌적한 사용 경험**을 보장하는 권장 버전 (본 문서 기준) |

### 2. 실제 프로젝트 설정 vs 권장 스펙

| 플랫폼 | 코드 설정 (최소 지원) | 문서 권장 스펙 | 판정 |
|--------|----------------------|----------------|------|
| Android | minSdk 24 (Android 7.0) | Android 10 (API 29) | ✅ 권장이 3세대 높음 |
| iOS | deploymentTarget 15.1 | iOS 17 | ✅ 권장이 2세대 높음 |

### 3. 프레임워크 공식 요구사항

| 프레임워크 | Android 요구사항 | iOS 요구사항 |
|------------|------------------|--------------|
| React Native 0.79.x | SDK 24+ | iOS 13+ |
| Expo SDK 53 | SDK 24+ | iOS 15.1+ |

**결론**: 권장 스펙(Android 10, iOS 17)은 프레임워크 공식 요구사항보다 2~3세대 위이며, 실제 프로젝트 minSdk/deploymentTarget 설정보다도 높아 **앱 구동에 충분한 여유**가 있음.

### 4. 권장 스펙 선정 기준

- **95% 커버리지 원칙**: 전체 사용자의 95% 이상이 사용하는 OS 버전 기준
- **2019-2020년 출시 기기 기준**: 중급 이상 스마트폰에서 쾌적한 UX 보장
- **벤더 중립적 하드웨어 스펙**: 특정 제조사에 의존하지 않는 범용 기준
- **RAM 4GB 기준**: 2020년 보급형 스마트폰 기본 탑재 용량 (Galaxy A51, Redmi Note 9, iPhone 11 등)

### 5. 웹 브라우저 버전 기준

| 브라우저 | 권장 버전 | 출시 시점 | 비고 |
|----------|-----------|-----------|------|
| Chrome | 100+ | 2022년 3월 | 자동 업데이트 |
| Edge | 100+ | 2022년 4월 | 자동 업데이트 |
| Firefox | 100+ | 2022년 5월 | 자동 업데이트 |
| Safari | 15+ | 2021년 9월 | iOS 15 / macOS Monterey 이상 |

**선정 근거:**
- **버전 100 기준**: 2022년 출시 버전으로, 현재 기준 약 3년 전 버전
- **자동 업데이트 정책**: Chrome, Edge, Firefox는 자동 업데이트가 기본이므로 대다수 사용자가 최신 버전 사용
- **Safari OS 종속성**: Safari는 OS 버전에 종속되어 별도 기준 적용 (iOS 15/macOS Monterey = 2021년)
- **95% 커버리지**: 브라우저 자동 업데이트 특성상 3년 전 버전이면 사실상 99% 이상 커버
