## 변경 개요

마음약국(CBT) 기능의 상태 관리 아키텍처를 전면 리팩토링.
**서버 API 응답 + AsyncStorage 6~7개 키의 이중 소스 → Zustand 단일 스토어 (Single Source of Truth)** 로 전환.

### 변경 목적
1. 서버/로컬 상태 합치기 파이프라인의 복잡도 제거 (버그 온상)
2. session.tsx 1505줄 → 3개 파일로 분리 (인수인계 가독성)
3. index.tsx 1033줄 → ~310줄로 축소 (변환 파이프라인 제거)
4. finishSession 6곳 호출 → 1곳으로 통합 (중복 호출 방지)
5. clearAllCBTStorage 누락 키 문제 해결 (resetAll로 완전 초기화)

## 내부 아키텍처 변경

### 신규 파일

| 파일 | 역할 |
|------|------|
| `src/store/cbt-store.ts` | Zustand + persist 기반 통합 상태 관리 |
| `src/features/cbt/lesson-derive.ts` | CbtState → UI 데이터 계산 순수 함수 |
| `src/features/cbt/sync-service.ts` | 서버 싱크 (fire-and-forget) |
| `src/features/cbt/hooks/use-session-flow.ts` | 세션 상태 + 스텝 진행 로직 |
| `src/features/cbt/hooks/use-session-completion.ts` | 완료/CLOSING/분기 처리 |

### 핵심 변경

```
이전 구조:
  서버 API (progress_status) + AsyncStorage 6~7개 키
  → applyClientSideLocking → applyLocalSessionOrder → applyClientSideLessonLocking
  → transformApiToUi
  = 복잡한 합치기 파이프라인

이후 구조:
  서버 API (콘텐츠만) + CbtStore (Zustand, 단일 AsyncStorage 키)
  → deriveLessonsForUI (순수 함수 1개)
  = 단순한 파생 계산
```

### 삭제 대상 (향후 테스트 마이그레이션 후)

| 파일/함수 | 대체 |
|-----------|------|
| `storage.ts` 전체 | `cbt-store.ts` (Zustand persist) |
| `progress-utils.ts` | `lesson-derive.ts` (deriveProgressValue) |
| `applyClientSideLocking()` | `deriveSessionsForUI()` |
| `applyLocalSessionOrder()` | CbtStore.lessonPriority |
| `applyClientSideLessonLocking()` | `isLessonLocked()` |
| `shouldIgnoreLockedSessions()` | 제거 (항상 true) |
| `transformApiToUi()` | `deriveLessonsForUI()` |
| `clearAllCBTStorage()` | `cbtStore.resetAll()` |

## 마이그레이션

기존 사용자의 AsyncStorage 데이터를 새 Zustand store로 자동 변환.
- `migrateLegacyCbtStorage()` 함수가 앱 시작 시 1회 실행
- 마이그레이션 완료 후 기존 키 삭제
- 실패해도 앱 동작에 영향 없음 (새 store가 빈 상태로 시작)
